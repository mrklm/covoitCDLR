    Aide

    Cette carte a pour but d'aider l'équipe technique du festival Chalon dans la rue à organiser des covoiturages.

    Cliquez sur votre nom dans la liste, puis indiquez si vous proposez un covoit ou si vous cherchez une place.

    Les trajets proposés sont affichés sur la carte avec une ligne colorée.

    Les trajets dont la date est passée ne sont plus affichés sur la carte. Les messages actifs défilent dans le bandeau sous l'en-tête.

    Le sélecteur au-dessus de la carte permet d'afficher les allers ou les retours.

    Pour l'aller, le départ est la ville de la personne par défaut et l'arrivée est Chalon-sur-Saône. Pour le retour, le départ est Chalon-sur-Saône et l'arrivée est la ville de la personne par défaut. Les villes sont modifiables et peuvent être changées.

    Si une ville manque dans une liste, choisissez + Ajouter une ville..., recherchez-la avec son code postal si besoin, puis validez le résultat.

    Les données sont enregistrées automatiquement et partagées avec les autres utilisateurs.

    Pour les geeks

    covoitCDLR est une application TypeScript côté client construite avec Vite. La carte est rendue avec Leaflet et les tuiles OpenStreetMap.

    Les données relatives aux technicien-ne-s sont lues dans Supabase via une fonction RPC protégée par mot de passe. Les trajets, messages et villes ajoutées sont stockés dans Supabase puis synchronisés en temps réel entre les appareils.

    Le navigateur conserve aussi une copie locale pour garder une interface réactive. Si Supabase n'accepte pas une annonce, l'application prévient que le message est visible seulement sur l'appareil utilisé.

    Les trajets passés sont masqués automatiquement selon la date du jour. Les lignes de trajet utilisent les coordonnées GPS des villes sélectionnées, et les villes manquantes peuvent être ajoutées via l'API publique geo.api.gouv.fr.
